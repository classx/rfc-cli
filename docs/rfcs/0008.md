## Problem

Текущая проверка drift в `doctor` основана на `mtime` файлов. После `git clone` и `checkout` времена модификации не отражают историю изменений: порядок записи файлов может отличаться, и `mtime` будет ложным. В результате `doctor` может выдавать ошибки drift, даже если код соответствует принятому RFC.

Нужно сделать проверку drift устойчивой к клонам и чек-аутам.

## Goal

Добавить стратегию drift на основе времени последнего git-коммита и сделать её стратегией по умолчанию. При этом должна сохраниться возможность использовать старую mtime-логику.

## Requirements

1. Добавить флаг `--drift=git|mtime` к команде `rfc-cli doctor`.
2. Значение по умолчанию: `--drift=git`.
3. Стратегия `git` должна сравнивать время последнего коммита для RFC-файла и каждого связанного файла (`links`).
4. Drift фиксируется, если последний коммит linked-файла новее последнего коммита RFC-файла.
5. Стратегия `mtime` должна сохранить текущее поведение без изменений.
6. При отсутствии git-репозитория или ошибки получения commit time команда должна:
   - вернуть понятную ошибку, **или**
   - (опционально) предложить использовать `--drift=mtime`.
7. Формат сообщений drift не меняется: `code drift: <path> modified after RFC acceptance`.
8. Поведение остальных проверок `doctor` не меняется.

## Design

### CLI

Расширить команду `doctor`:

```rust
/// Run project-wide RFC health diagnostics
Doctor {
    /// Threshold in days for detecting stale drafts (default: 30)
    #[arg(long, default_value = "30")]
    stale_days: u64,

    /// Drift detection strategy (default: git)
    #[arg(long, value_enum, default_value = "git")]
    drift: DriftStrategy,
}
```

Добавить enum:

```rust
#[derive(clap::ValueEnum, Clone, Copy)]
pub enum DriftStrategy {
    Git,
    Mtime,
}
```

### Логика drift

В `doctor` определить стратегию:

- `Git`:
  - RFC time: `git log -1 --format=%ct -- docs/rfcs/NNNN.md`
  - Link time: `git log -1 --format=%ct -- <link>`
  - Drift если `link_time > rfc_time`

- `Mtime`:
  - Сохранить текущую реализацию на основе `fs::metadata().modified()`.

### Обработка ошибок git

Варианты (предпочтительно выбрать один в реализации):

1. **Ошибка с подсказкой**  
   Если git недоступен или файл не отслеживается:
   `Error: git commit time unavailable for <path>. Try --drift=mtime`

2. **Fallback**  
   Если git недоступен, автоматически использовать `mtime` с предупреждением.

## Alternatives

1. **Оставить только mtime**  
   Не решает проблему ложных drift после `git clone`.

2. **Сравнивать хеши linked-файлов**  
   Надежно, но требует хранить хеши для каждого linked-файла в индексе.

3. **Использовать git blame для каждой строки**  
   Слишком тяжелая проверка, не подходит для быстрой диагностики.

## Migration

Миграция не требуется. Обновляется только логика `doctor` и CLI.

Шаги:
1. Добавить `DriftStrategy` и флаг `--drift`.
2. Реализовать `git`-стратегию.
3. Обновить тесты.

## Test Plan

- `doctor --drift=mtime` использует текущую логику и не меняет вывод.
- `doctor` по умолчанию (`--drift=git`) использует git-стратегию.
- Если linked-файл имеет более новый коммит, чем RFC-файл — фиксируется drift.
- Если git недоступен — корректная ошибка/подсказка.
- Все остальные проверки `doctor` работают как раньше.
