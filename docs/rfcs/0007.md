## Problem

Сейчас в проекте нет формализованного файла с контекстом и правилами для AI-ассистентов. В результате:
- инструкции по RFC-процессу, статусам и обязательным проверкам разрознены;
- нет единого "источника правды" для подсказок AI (локально или в CI);
- новые участники/ассистенты не понимают, где искать правила.

Нужна команда, которая будет генерировать/обновлять файл `AGENTS.md` с понятным описанием RFC-процесса и CLI-команд.

## Goal

Добавить новую команду `rfc-cli agents`, которая:
1. генерирует файл `AGENTS.md` на английском языке в корне проекта, если он отсутствует;
2. если файл уже существует — добавляет в него секцию с актуальными правилами (без затирания пользовательского контента);
3. использует единый шаблон и стабильный маркер секции для обновления;
4. поддерживает идемпотентность (повторный запуск не дублирует секцию).

## Requirements

1. Команда: `rfc-cli agents`.
2. Путь файла: `<project_root>/AGENTS.md`.
3. Содержимое `AGENTS.md` и секции должно быть на английском языке.
4. Если файла нет — создать новый файл с секцией.
5. Если файл есть — вставить секцию:
   - если секция отсутствует — добавить в конец файла;
   - если секция уже есть — заменить содержимое секции на актуальное.
6. Секция должна иметь явные маркеры:
   - начало: `<!-- rfc-cli:agents:start -->`
   - конец: `<!-- rfc-cli:agents:end -->`
7. Секция должна включать:
   - краткое описание RFC-процесса;
   - статусный цикл и допустимые статусы;
   - основные команды (`init`, `new`, `list`, `set`, `check`, `doctor`, `reindex`);
   - обязательные правила (например: "нет кода без RFC", "accepted RFC не править напрямую").
8. Вывод команды:
   - при создании файла: `Generated AGENTS.md ✅`
   - при обновлении: `Updated AGENTS.md ✅`
9. Код выхода:
   - `0` при успехе;
   - `1` при ошибке (невозможно записать файл, нет прав и т.д.).

## Design

### CLI

Добавить в `src/cli.rs` подкоманду:

```rust
/// Generate AI assistant instructions (AGENTS.md)
Agents
```

### Команда

Файл: `src/commands/agents.rs`

```rust
pub fn execute(project_root: &Path) -> Result<(), String>
```

Алгоритм:
1. Определить путь `AGENTS.md` в корне проекта.
2. Сформировать строку секции с маркерами `<!-- rfc-cli:agents:start -->` и `<!-- rfc-cli:agents:end -->`.
3. Если файл не существует — создать новый с этой секцией.
4. Если файл существует:
   - если маркеры найдены — заменить содержимое между ними;
   - если маркеров нет — добавить секцию в конец файла с пустой строкой перед ней.
5. Записать файл.
6. Вывести сообщение (Generated/Updated).

### Содержимое секции

Минимальный шаблон:

```
<!-- rfc-cli:agents:start -->
# AI Agent Guide (rfc-cli)

## RFC Process
- No code without an RFC.
- Accepted/implemented RFCs are immutable: create a new RFC to change decisions.

## Status Lifecycle
draft → review → accepted → implemented
                ↓            ↓
           deprecated    superseded

Allowed statuses: draft, review, accepted, implemented, superseded, deprecated.

## Commands
- rfc-cli init
- rfc-cli new "<title>"
- rfc-cli set <N> <status> [--by N]
- rfc-cli list
- rfc-cli check [N]
- rfc-cli doctor [--stale-days N]
- rfc-cli reindex

## Validation
- Always run: rfc-cli check
- For repo health: rfc-cli doctor

<!-- rfc-cli:agents:end -->
```

## Alternatives

1. **Статический AGENTS.md, поддерживаемый вручную**  
   Недостаток: легко устаревает, нет гарантии синхронизации с CLI.

2. **Генерация через `rfc-cli doctor` или `check`**  
   Недостаток: смешение ответственности команд.

3. **Хранить инструкции только в README**  
   Недостаток: AI-агенты часто ищут именно `AGENTS.md`/`AGENT.md` как стандартный файл.

## Migration

Миграция не требуется. Добавляется новая команда и файл.

Шаги:
1. Добавить команду `agents` в CLI и dispatch.
2. Реализовать генерацию/обновление `AGENTS.md`.
3. Написать тесты: создание файла, обновление с маркерами, добавление без маркеров.
4. Привязать файлы к RFC: `rfc-cli link 0007 src/commands/agents.rs src/cli.rs src/main.rs`.

## Test Plan

- `agents` создаёт новый `AGENTS.md` при отсутствии файла.
- `agents` обновляет существующую секцию между маркерами.
- `agents` добавляет секцию в конец, если маркеров нет.
- Повторный запуск не дублирует секцию.
- Вывод сообщений соответствует требованиям.
