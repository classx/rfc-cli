# rfc-cli

CLI-инструмент для управления RFC (Request for Comments) документами в проекте.

RFC — это формальные спецификации, которые описывают проектные решения **до** начала реализации. Это позволяет:

- фиксировать архитектурные решения и их обоснование
- обсуждать дизайн до написания кода
- давать AI-ассистентам точный контекст вместо расплывчатых запросов
- сохранять историю решений и отвергнутых альтернатив

## Установка

```sh
cargo build --release
```

Бинарный файл будет в `target/release/rfc-cli`.

## Быстрый старт

```sh
# Инициализация структуры RFC в проекте
rfc-cli init

# Создание нового RFC
rfc-cli new "название идеи"

# Список всех RFC
rfc-cli list

# Просмотр RFC
rfc-cli view 0001
```

## Команды

| Команда | Описание |
|---------|----------|
| `rfc-cli init` | Создаёт каталог `docs/rfcs/` и индексный файл |
| `rfc-cli new <title>` | Создаёт новый RFC из шаблона с автоматической нумерацией |
| `rfc-cli list [--status <status>]` | Выводит таблицу RFC с возможностью фильтрации по статусу |
| `rfc-cli view <number>` | Выводит содержимое RFC в терминал |
| `rfc-cli edit <number>` | Открывает RFC в `$EDITOR` |
| `rfc-cli status <number>` | Выводит текущий статус RFC |
| `rfc-cli set <number> <status>` | Меняет статус RFC с проверкой допустимых переходов |
| `rfc-cli link <number> <path>` | Связывает RFC с файлом исходного кода |
| `rfc-cli unlink <number> <path>` | Удаляет связь RFC с файлом |
| `rfc-cli deps <number>` | Выводит дерево зависимостей RFC |
| `rfc-cli check [<number>]` | Валидирует формат RFC (frontmatter, секции, ссылки) |
| `rfc-cli doctor` | Диагностика здоровья связи между RFC и кодовой базой |
| `rfc-cli reindex` | Полностью перестраивает индексный файл |

## RFC-процесс

### Правила

1. **Нельзя писать код без RFC** — любое нетривиальное изменение начинается с RFC
2. **RFC можно нарушить только через новый RFC** — прямое редактирование принятых RFC запрещено
3. **Статусы обязательны** — каждый RFC проходит через определённый жизненный цикл

### Жизненный цикл

```
draft → review → accepted → implemented
                    ↓            ↓
               deprecated    superseded
```

| Статус | Значение |
|--------|----------|
| `draft` | Черновик, идёт написание |
| `review` | Документ готов к обсуждению |
| `accepted` | Решение утверждено, можно реализовывать |
| `implemented` | Реализация завершена |
| `superseded` | Заменён новым RFC |
| `deprecated` | Устарел или отменён без замены |

### Формат RFC-документа

Каждый RFC — это Markdown-файл с YAML frontmatter:

```yaml
---
title: "RFC-0001: название"
status: draft
dependencies: []
superseded_by: null
links: []
---

## Проблема
## Задача
## Дизайн
## Альтернативы
## Голосование
## Миграция
```
